# 蚁群优化算法 (Ant Colony Optimization)

## 算法概述

蚁群优化算法是一种模拟蚂蚁觅食行为的群体智能算法，通过蚂蚁在路径上释放和感知信息素来寻找最优路径。

## 核心原理

1. **信息素机制**：蚂蚁在路径上释放信息素，后续蚂蚁根据信息素浓度选择路径
2. **正反馈机制**：较短路径上的信息素积累更快，吸引更多蚂蚁
3. **负反馈机制**：信息素逐渐蒸发，避免算法陷入局部最优
4. **并行搜索**：多只蚂蚁同时搜索，增强全局搜索能力

## 算法流程

```
初始化信息素 → 放置蚂蚁 → 蚂蚁构建路径 → 更新信息素 → 蒸发信息素 → 重复直到满足停止条件
```

## 代码结构

- `AntColonyAnt.java`: 蚂蚁类，实现单只蚂蚁的路径构建行为
- `AntColonyOptimization.java`: 蚁群优化主类，实现算法流程控制
- `TSPExample.java`: TSP问题应用示例

## 经典应用

### 1. 组合优化问题
- **旅行商问题 (TSP)**: 寻找访问所有城市的最短路径
- **车辆路径问题 (VRP)**: 多车辆配送路径优化
- **二次分配问题 (QAP)**: 设施布局优化

### 2. 图论问题
- **最短路径问题**: 在网络中寻找最优路径
- **最大流问题**: 网络最大流量计算

### 3. 工程应用
- **通信网络路由**: 数据包路由选择
- **电力系统规划**: 电网优化设计
- **生产线调度**: 制造系统优化

### 4. 其他领域
- **图像处理**: 图像分割和特征提取
- **数据挖掘**: 聚类分析和关联规则挖掘
- **机器人路径规划**: 多机器人协调路径规划

## 参数说明

- **蚂蚁数量 (numAnts)**: 影响算法并行性，通常与城市数量相当
- **信息素因子 (alpha)**: 控制信息素重要程度，通常1.0-5.0
- **启发式因子 (beta)**: 控制距离启发信息重要程度，通常1.0-5.0
- **蒸发率 (rho)**: 信息素蒸发速度，通常0.1-0.5
- **信息素常量 (Q)**: 信息素释放量，通常10-1000

## 优势

- 全局搜索能力强
- 正反馈机制加速收敛
- 分布式计算，易于并行化
- 对问题结构要求低
- 鲁棒性强

## 局限性

- 参数选择敏感
- 收敛速度较慢
- 易陷入局部最优
- 对动态环境适应性差

## 使用建议

1. 合理设置信息素参数 (alpha, beta)
2. 选择合适的蒸发率 (rho)
3. 蚂蚁数量与问题规模匹配
4. 结合局部搜索算法使用
5. 采用多种初始化策略

## 数学基础

### 状态转移规则
```
P_{ij}^k = (τ_{ij}^α × η_{ij}^β) / Σ(τ_{ik}^α × η_{ik}^β)
```

其中：
- τ_{ij}: 边(i,j)上的信息素浓度
- η_{ij}: 边(i,j)的启发信息 (通常为距离的倒数)
- α: 信息素重要程度参数
- β: 启发信息重要程度参数

### 信息素更新规则
```
τ_{ij} ← (1-ρ)×τ_{ij} + ΣΔτ_{ij}
Δτ_{ij} = Q/L_k  (如果蚂蚁k在路径中使用了边(i,j))
```
