# 禁忌搜索算法 (Tabu Search)

## 算法概述

禁忌搜索是一种元启发式算法，通过维护禁忌表来避免搜索过程中重复访问最近访问过的解，从而避免算法陷入局部最优。

## 核心原理

1. **禁忌机制**：记录最近访问的解，禁止短期内重复访问
2. **特赦准则**：即使禁忌解，如果足够好也可以接受
3. **邻域搜索**：在当前解的邻域中搜索更好解
4. **长期记忆**：通过频率记忆指导搜索方向

## 算法流程

```
初始化当前解和禁忌表 → 重复以下步骤直到满足停止条件:
  生成候选解集合 → 选择最好的非禁忌候选解 → 更新禁忌表 → 更新当前解
```

- `TabuSearch.java`: 禁忌搜索算法主类，实现TSP问题的禁忌搜索

## 经典应用

### 1. 组合优化
- **旅行商问题 (TSP)**: 寻找最短路径
- **车辆路径问题 (VRP)**: 多车辆配送优化
- **二次分配问题 (QAP)**: 设施布局优化

### 2. 调度问题
- **作业车间调度**: 生产调度优化
- **机组调度**: 电力系统调度
- **员工排班**: 人力资源优化

### 3. 图论问题
- **图着色问题**: 最小着色数
- **最大团问题**: 最大完全子图
- **最小顶点覆盖**: 图论优化

### 4. 工程应用
- **电路设计**: VLSI布局布线
- **通信网络**: 网络拓扑优化
- **供应链管理**: 库存和配送优化

### 5. 其他领域
- **金融投资**: 投资组合优化
- **蛋白质设计**: 分子结构优化
- **机器学习**: 特征选择优化

## 参数说明

- **禁忌期限 (tabuTenure)**: 禁忌移动保持的时间，通常5-20
- **最大迭代次数 (maxIterations)**: 算法停止条件，通常100-1000
- **候选解数量**: 每次迭代生成的候选解数
- **邻域结构**: 定义解空间的邻域关系

## 禁忌表管理

### 禁忌条件
- 最近访问的解不可立即访问
- 禁忌期限控制禁止访问的时间
- 特赦准则允许访问特别好的禁忌解

### 禁忌对象
- 移动 (moves)
- 解的特征 (attributes)
- 频率信息 (frequencies)

## 邻域结构

### 2-opt邻域 (TSP)
- 交换路径中的两条边
- 适用于路径优化问题

### 插入邻域
- 移除一个元素，插入到其他位置
- 适用于排序问题

### 交换邻域
- 交换两个元素的位置
- 通用邻域结构

## 优势

- 能有效跳出局部最优
- 搜索过程更智能
- 对初始解不敏感
- 适用于大规模问题
- 收敛速度较快

## 局限性

- 参数选择困难
- 禁忌表管理复杂
- 对邻域结构敏感
- 难以理论分析

## 使用建议

1. **参数设置**:
   - 禁忌期限: 通常设为问题规模的5%-20%
   - 候选解数量: 通常为邻域大小的10%-50%
   - 迭代次数: 根据问题复杂度和时间限制

2. **邻域设计**:
   - 选择合适的邻域结构
   - 邻域大小适中
   - 保证邻域连通性

3. **禁忌表设计**:
   - 选择合适的禁忌对象
   - 设置合理的禁忌期限
   - 实现特赦准则

## 高级特性

### 长期记忆
- 频率记忆: 记录解组件的使用频率
- 质量记忆: 记录解组件的质量信息
- 影响搜索方向和强度

### 多样化策略
- 重新启动: 从新初始解开始搜索
- 扰动机制: 强制改变搜索方向
- 多级邻域: 使用不同粒度的邻域

### 并行禁忌搜索
- 多起点搜索
- 合作搜索
- 竞争搜索

## 数学基础

禁忌搜索不保证收敛到全局最优，但实践证明在许多问题上表现优秀。
